# 05_AGENT_HANDOFF

## 역할

이 문서는 일반 작업 원칙이 아니라, 작업을 나누거나 넘길 때 필요한 맥락을 정리한다.
상위 작업 원칙, 금지 사항, 문서 갱신 규칙은 루트 `AGENTS.md`를 따른다.

## 핸드오프가 필요한 경우

- 한 번의 변경으로 끝내기 어려운 구조 변경이 생겼을 때
- 홈, 콘텐츠 모델, 라우트 변경이 함께 묶여 순차 작업이 필요할 때
- 현재 작업자가 결정하지 못할 제품 판단이 남아 있을 때
- 정리 작업과 기능 작업을 분리해야 할 때

## 허용 작업 예시

- HeroSlider를 컴포넌트로 분리
- ProductCard 공통화
- NoticeSection를 데이터 기반 렌더링 구조로 변경
- TrustSection 스타일 정리

## 핸드오프 시 반드시 남길 내용

- 무엇을 바꿨는지
- 어떤 문서를 같이 갱신했는지
- 아직 남아 있는 결정 사항이 무엇인지
- 다음 작업자가 어디서부터 보면 되는지

## 핸드오프 단위

- 가능하면 섹션 단위, 라우트 단위, 데이터 모델 단위로 나눈다
- 홈 구조 변경은 섹션 스펙 확인과 구현 변경을 한 묶음으로 본다
- content shape 변경은 문서 갱신과 타입 변경을 같은 단계로 묶는다
