DESIGN_RULES_EXTRACTED.md

이 문서는 현재 코드베이스(`docs`, `src/styles`, `src/components`)에서 실제로 사용 중인 UI 패턴을 기준으로 **디자인 규칙을 추출**한 정리본입니다.

## 1) 디자인 톤 & 방향성
- 시장 공식몰형 톤을 유지한다.
- 점포/상품을 병행 노출하는 정보구조를 우선한다.
- 밝은 배경 위에 포인트 컬러(Primary/Secondary)로 강조한다.
- 이벤트/혜택, 지도 인터랙션이 눈에 띄도록 대비를 설계한다.

## 2) 컬러 시스템 (토큰 고정 사용)
- `--primary`: `#e8261a` (핵심 CTA, 강조)
- `--primary-light`: `#fdecea` (보조 배경)
- `--secondary`: `#2e7d32` (보조 강조, 활성 상태)
- `--secondary-light`: `#e8f5e9`
- `--bg`: `#ffffff` (기본 배경)
- `--card`: `#ffffff` (카드 배경)
- `--muted`: `#f5f5f5` (중립 배경)
- `--muted-fg`: `#6b7280` (보조 텍스트)
- `--border`: `#e5e7eb` (경계선)
- `--fg`: `#111827` (기본 텍스트)

### 컬러 적용 규칙
- 본문/핵심 텍스트는 `--fg`를 기본으로 사용한다.
- 설명/날짜/메타 정보는 `--muted-fg`를 사용한다.
- 버튼·핵심 배지·강조 라벨은 `--primary`를 우선한다.
- 상태 전환(예: map marker active)에는 `--secondary`를 사용한다.

## 3) 레이아웃 규칙
- 컨테이너 최대 너비: `1200px`
- 가로 패딩: `20px`
- 섹션 상단 여백: `60px`
- 카드/패널 기본 radius는 `12px`~`20px` 사이 패턴을 유지한다.

### 공통 배치 원칙
- 섹션은 `.section` 단위로 분리해 시각적 블록을 명확히 나눈다.
- 섹션 헤더는 `label -> title -> action` 구조를 사용한다.
- 헤더/네비는 경계선(`--border`)으로 정보영역을 분할한다.

## 4) 타이포그래피 규칙
- 기본 폰트: `'Apple SD Gothic Neo', 'Noto Sans KR', sans-serif`
- 본문 line-height: `1.5`
- 제목 계층:
  - 섹션 타이틀: `28px / 900`
  - 히어로 타이틀: `42px` (반응형에서 축소)
- 보조 정보는 `11px~14px`의 작은 크기와 `--muted-fg`를 사용한다.

## 5) 컴포넌트 규칙

### Button
- 기본 버튼은 pill 느낌이 아닌 라운드 직사각형(`10px`)을 사용한다.
- 기본 배경은 `--primary`, 텍스트는 흰색, 굵기 `700`을 유지한다.

### Card
- 카드 배경은 흰색, `1px solid --border`, radius는 토큰 또는 섹션 패턴에 맞춘다.
- hover 인터랙션이 필요할 경우 `translateY` + 부드러운 그림자 패턴을 사용한다.

### Badge
- 크기는 작고(`10px`), 용도별 색상 변형(`primary`, `secondary`, `neutral`)을 사용한다.

### Form/Search
- 검색 입력은 높은 라운드(`999px`)와 primary 계열 border로 구성한다.
- 검색 버튼은 원형에 primary 배경으로 시인성을 확보한다.

## 6) 페이지별 패턴 규칙

### Hero
- 고정 높이 배너(데스크톱 기준 `460px`)를 사용한다.
- 슬라이드 전환은 `0.6s ease`를 기본으로 한다.
- 타이틀/서브타이틀/CTA의 3단 정보 우선순위를 유지한다.

### Store + Product Feed
- 점포 정보 + 상품 리스트를 한 그룹 카드로 묶는다.
- 데스크톱에서는 `좌(점포) / 우(상품)` 2열, 모바일에서는 1열로 전환한다.
- 상품카드는 이미지(정사각), 카테고리, 상품명, 가격/할인, 태그 순서를 유지한다.

### Map
- 지도 영역은 라운드 패널 + 부드러운 배경 그라데이션을 사용한다.
- 마커는 primary 기본, active는 secondary로 구분한다.
- 모바일에서는 하단 시트(`bottom sheet`)로 상세 정보를 분리 제공한다.

## 7) 반응형 규칙
- Breakpoint 960px:
  - 헤더 wrap 허용
  - 복수 그리드(신뢰, 푸터, 지도, 점포/상품 그룹)를 1~2열로 축소
- Breakpoint 640px:
  - 상단 보조 바 숨김
  - 섹션 타이틀 축소
  - 주요 리스트를 1열로 단순화

## 8) 구현 가이드 (실무 적용)
- 새 UI를 만들 때 먼저 CSS 토큰(`--primary`, `--border`, `--radius` 등)만으로 구현한다.
- 임의 색상/간격 하드코딩은 지양하고 기존 스케일을 재사용한다.
- 공통 UI는 `src/components/ui/*` + 공통 클래스(`.button`, `.card`, `.container`) 조합을 우선한다.
- 신규 섹션도 기존 `SectionHeader` 패턴을 기본 골격으로 사용한다.
